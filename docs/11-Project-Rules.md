# Inzane's Food Project Rules
Version: 1.0

---

# Project Philosophy

The website should represent a premium Asian comfort food restaurant while demonstrating modern restaurant technology.

Every design decision should support the following goals:

• Build customer trust

• Increase online orders

• Encourage reservations

• Demonstrate AI-powered restaurant automation

• Maintain premium branding

• Deliver a fast and enjoyable user experience

Never sacrifice usability for visual effects.

---

# Brand Rules

Always use the official brand assets.

Never redesign the logo.

Never replace the typography without approval.

Never change the official brand colors.

Always maintain consistent spacing.

Always maintain consistent component styles.

Never create additional logo variations.

Use consistent naming.

---

# Food Photography Rules

Only use:

Professional food photography

Warm lighting

High resolution

Natural colors

Realistic presentation

Use uploaded official food images whenever available.

If an official image is unavailable:

Use a premium placeholder.

Replace the placeholder when the official image becomes available.

Never use cartoon illustrations.

Never use low-quality images.

Never stretch food photography.

---

# Navigation Rules

Sticky navigation

Transparent hero

White navigation after scrolling

Always highlight the active page.

Logo always returns to Home.

Order Now button always visible.

# Content Rules

Use clear language.

Avoid jargon.

Use friendly and welcoming tone.

Never use placeholder text in production.

Keep messaging consistent with the brand voice.

---

# Menu Rules

Display official prices.

Do not invent menu items.

Do not invent promotions.

Use the official menu database.

Show placeholders only for missing images.

---

# AI Rules

AI should:

Be polite

Be concise

Be accurate

Recommend relevant menu items

Answer FAQs

Assist with reservations

Assist with ordering

Escalate complex issues to staff

Never invent restaurant policies.

Never guess unavailable menu items.

Never promise unavailable services.

Restrict admin access.
